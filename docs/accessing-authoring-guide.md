# How to open the Authoring Guide (exact steps)

The Authoring Guide lives inside the CursorTest repo at: `docs/authoring-guide.md`.
Use any ONE of the methods below.

## Method 1: Open in GitHub (fastest)
1) Open this link in your browser:
   - Authoring Guide: [docs/authoring-guide.md](https://github.com/joekarschnikbiberk/CursorTest/blob/main/docs/authoring-guide.md)
2) Read it online.
3) Optional: Click “Raw” (top-right of the file) to copy/paste the template.

## Method 2: Open in Cursor (local)
1) Open Cursor.
2) File → Add Folder to Workspace…
3) Select: `C:\Users\Joe.Karschnik\FrontEndQA\CursorTest`
4) In the left sidebar, open `docs` → `authoring-guide.md`.

## Method 3: Open locally (File Explorer + any editor)
1) Open File Explorer and go to:
   - `C:\Users\Joe.Karschnik\FrontEndQA\CursorTest\docs\authoring-guide.md`
2) Double-click to open in your default editor (Cursor/VS Code/Notepad).

## Method 4: Clone the repo first (if you don’t have it)
1) Open a terminal (PowerShell).
2) Run:
   ```powershell
   cd C:\Users\Joe.Karschnik\FrontEndQA
   git clone https://github.com/joekarschnikbiberk/CursorTest.git
   ```
3) Then use Method 2 or 3.

## Where to paste acceptance criteria
- Put your acceptance criteria at the top of your feature draft (or PR description).
- Then open the Authoring Guide and copy the short template.
- Fill it out using the exact step phrases shown (don’t invent synonyms).

## If you get stuck
- Can’t find the file? Use Method 1 (GitHub link) and bookmark it.
- Repo missing in Cursor? Use File → Add Folder to Workspace… → pick `CursorTest`.
- Still blocked? Ping the QA team lead with this link:
  - [docs/authoring-guide.md](https://github.com/joekarschnikbiberk/CursorTest/blob/main/docs/authoring-guide.md)
